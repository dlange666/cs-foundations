# 前缀和（Prefix Sum）

前缀和用于快速计算区间和。  
定义 `pre[i]` 表示前 `i` 个元素之和（不包含 `nums[i]`），则区间 `[l, r]` 和为：

`sum(l, r) = pre[r + 1] - pre[l]`

## 1. 一维前缀和模板

```python
def build_prefix(nums):
    pre = [0] * (len(nums) + 1)
    for i in range(len(nums)):
        pre[i + 1] = pre[i] + nums[i]
    return pre

def range_sum(pre, l, r):
    return pre[r + 1] - pre[l]
```

构建复杂度 `O(n)`，单次查询 `O(1)`。

## 2. 前缀和 + 哈希表

用于“查找和为 K 的子数组个数”等问题。

思路：

- 设当前前缀和为 `s`。
- 若之前出现过 `s - k`，说明存在一个区间和为 `k`。
- 用哈希表记录每个前缀和出现次数。

```python
def subarray_sum(nums, k):
    count = 0
    s = 0
    freq = {0: 1}
    for x in nums:
        s += x
        count += freq.get(s - k, 0)
        freq[s] = freq.get(s, 0) + 1
    return count
```

时间复杂度 `O(n)`，空间复杂度 `O(n)`。

## 3. 解题步骤

1. 判断题目是“区间和查询”还是“满足某和关系的区间计数”。
2. 区间查询用数组前缀和；计数/查找用“前缀和 + 哈希”。
3. 统一前缀和定义为长度 `n + 1`，减少边界判断。
4. 注意负数场景下，滑动窗口不适用，优先前缀和 + 哈希。

## 4. 易错点

- 忘记初始化 `freq = {0: 1}`，导致漏掉从下标 `0` 开始的区间。
- 把区间公式写成 `pre[r] - pre[l]`（少算右端点）。
- 计数题误写成只记录“是否存在”，导致结果偏小。
